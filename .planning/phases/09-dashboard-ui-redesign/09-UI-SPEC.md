# Phase 9 — UI Design Contract

> Visual and interaction contract for the full presentational redesign of the GitChange local dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope guard:** This is a presentational redesign of an existing, working dashboard (`packages/dashboard/src/`). Component logic, API calls (`api/client.ts`, `snapshot.ts`), and store wiring (`store/drill.ts`, `store/tour.ts`, `store/workspace.ts`) MUST be preserved unchanged. This contract governs layout (JSX structure/class names), CSS, and copy only.

|-----------|-------|

## Information Architecture (redesign contract)

The current layout is a fixed two-column grid `lg:grid-cols-[minmax(0,22rem)_minmax(0,1fr)]` where every view is forced through a narrow 22rem left rail with dead space on the right. Replace with a **per-view layout** driven by the active tab. The top-level app frame is shared; the body region is owned by the active view.

### App frame (shared, all views)
- **Header bar** — full-width, sticky (`sticky top-0 z-20`). Contains: product title `GitChange` (left), then attribution badge + `HEAD {sha7}` + repo/workspace badge (right). Height 56px (`h-14`), horizontal padding `px-6`, bottom border `border-slate-800`, background `bg-slate-900/80` with `backdrop-blur`.
- **Primary tab nav** — moves OUT of the sidebar to a full-width horizontal bar directly under the header. Tabs: Timeline · Decisions · Open work · Tours · Graph. This is the top-level navigation for the whole app, not a sidebar widget.
- **Content max width** — raise from `max-w-7xl` to `max-w-[96rem]` (`1536px`) so wide views (Timeline, Graph) actually use available width. Center with `mx-auto`, page padding `px-6`.

### Per-view body layouts

| View | Layout contract |
|------|-----------------|
| **Timeline** | Full-width **era timeline strip on top** (spans the whole content width, min-height 8rem, its own card). Below it a **filter bar** (repo filter + commit filters) spanning full width. Below that a **two-pane split**: left = context rail `minmax(20rem, 24rem)` holding Index status + Era detail + File history (stacked, each collapsible-feeling as its own card); right = `minmax(0,1fr)` virtualized commit list that fills remaining width and height. On `< lg` the split collapses to single column (rail above list). This fixes both the crammed-sidebar and the dead-space defects: the timeline and commit list get full width, the auxiliary cards get an adequate rail, not a 22rem squeeze. |
| **Decisions** | Two-pane split: left = decisions list/browser `minmax(22rem, 28rem)`; right `1fr` = selected-decision evidence + drill. No global sidebar. |
| **Open work** | Two-pane split: left = open-threads list `minmax(22rem, 28rem)`; right `1fr` = selected migration thread timeline. |
| **Tours** | Two-pane split: left = tour picker + chapter nav `minmax(20rem, 24rem)`; right `1fr` = tour player / stop cards, comfortable reading measure (`max-w-3xl` for prose stop content inside the pane). |
| **Graph** | Full-bleed within content width. Graph canvas takes `1fr` height (min 32rem); a slim helper/legend strip sits above or below, not a 22rem sidebar. |

### Load / empty / error states (frame-level)
- Loading, empty (no index), and error states render **centered in the content region** (not jammed in a sidebar). Max readable width `max-w-md`, centered vertically in the body.

## Spacing Scale

Declared values (multiples of 4). Maps to Tailwind default scale (`gap-2`=8, `gap-4`=16, etc.).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge inner padding, chip padding |
| sm | 8px | Compact stack gaps, tab pill inner padding, metadata rows |
| md | 16px | Default card padding, gaps between fields, list-row padding |
| lg | 24px | Card-to-card gaps within a rail, section padding inside panels |
| xl | 32px | Major layout gaps (rail ↔ main pane), vertical rhythm between view regions |
| 2xl | 48px | Empty-state vertical padding, large breathing room around centered states |
| 3xl | 64px | Reserved for empty/hero states only |

**Exceptions:**
- Header height fixed at 56px (`h-14`) — a component dimension, not a spacing token.
- Interactive hit targets (tab buttons, filter controls, list rows) must be ≥ 32px tall; icon-only buttons ≥ 36px square.
- vis-timeline internal item margin stays at `{ item: 8 }` (library option, unchanged).

## Typography

Exactly 4 sizes, exactly 2 weights (regular 400, semibold 600). No other weights.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px (`text-2xl`) | 600 | 1.2 | Product title in header only |
| Heading | 18px (`text-lg`) | 600 | 1.25 | Card/panel titles ("Project timeline", "Index status", decision titles) |
| Body | 14px (`text-sm`) | 400 | 1.5 | Default reading text, list rows, panel prose, commit messages |
| Label/Meta | 12px (`text-xs`) | 400 (600 for active tab & badges) | 1.4 | Metadata (SHAs, dates, authors), tab labels, badge text, axis labels |

Rules:
- Never go below 12px. The current UI's "small type" defect is corrected by making Body 14px (not 12px) the default and reserving 12px for genuine metadata.
- Monospace SHAs/paths render at Label size (12px) or Body size (14px) depending on prominence; weight stays 400.
- Headings use `tracking-tight`. Body/meta use default tracking.

## Color

Dark theme. 60/30/10 split expressed in the existing slate + sky ladder. Improve contrast and hierarchy; do not introduce new hues beyond the semantic set already in the codebase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `slate-950` `#020617` | App background, main content background, commit-list background |
| Secondary (30%) | `slate-900` `#0f172a` (cards) / `slate-800` `#1e293b` (raised/hover surfaces, chips) | Cards, panels, rail surfaces, header, filter bar, nav bar |
| Border | `slate-800` `#1e293b` (default) / `slate-700` `#334155` (card borders, higher contrast) / `slate-600` `#475569` (hover/emphasis) | All card and divider borders. Standardize on `slate-800` for structural dividers and `slate-700` for card outlines. |
| Text | `slate-100` `#f1f5f9` (primary) / `slate-300` `#cbd5e1` (secondary body) / `slate-400` `#94a3b8` (metadata) / `slate-500` `#64748b` (de-emphasized hints only) | Text ladder. Raise the floor: metadata that must be readable uses `slate-400`, never `slate-500`. `slate-500` only for truly incidental hints. |
| Accent (10%) | `sky-400` `#38bdf8` (interactive/focus), `sky-500` `#0ea5e9` (selected fills), `sky-300` `#7dd3fc` (accent text/links) | See reserved-for list below. |
| Destructive | `red-400` `#f87171` text on `red-950/40` surface, `red-800` `#991b1b` border | Error alerts and destructive-action affordances only |

**Accent (sky) reserved for — explicit list, nothing else:**
1. Active/selected primary tab indicator (underline or filled pill).
2. Selected era item in the timeline and selected row highlight (ring/left-border) in lists.
3. Keyboard focus rings on all interactive elements (`ring-2 ring-sky-400 ring-offset-2 ring-offset-slate-950`).
4. Primary action buttons (e.g. "Load more", tour "Next stop") — background `sky-600`, text `slate-950`/white.
5. Inline evidence/drill links (`text-sky-300`, underline on hover).

Accent is NOT used for: card borders, ordinary body text, headings, decorative fills, or non-interactive backgrounds.

**Semantic state colors (status badges only — already in codebase, standardized):**

| State | Text | Surface | Border |
|-------|------|---------|--------|
| Complete / done | `emerald-200` | `emerald-950/40` | `emerald-700` |
| In progress | `amber-200` | `amber-950/40` | `amber-700` |
| Open / info | `sky-200` | `sky-950/40` | `sky-700` |
| Stale | `slate-300` | `slate-800/60` | `slate-600` |
| Error / blocked | `red-200` | `red-950/40` | `red-800` |

Retire incidental `violet`/`rose` one-offs; map them onto the five states above unless a specific badge semantics requires otherwise (note in the plan if so).

**Contrast requirement:** all body and metadata text must meet WCAG AA (≥ 4.5:1) against its surface; badge/label text ≥ 3:1. `slate-400` on `slate-900` and `slate-100` on `slate-950` both satisfy this; `slate-500` on `slate-900` does not for body text and is therefore restricted to non-essential hints.

## Component Treatment Contract

### Card / panel (the base surface for every list, detail panel, and status widget)
- Background `bg-slate-900`, border `border border-slate-700`, radius `rounded-lg` (8px), padding `p-4` (16px), header title at Heading size.
- Cards separated by `gap-6` (24px) within a rail, `gap-8` (32px) between major regions.
- A card header (title + optional badge/action) sits on its own row with `mb-3`; content follows.

### Scroll containers (fixes the "clipped header / first row" defect)
- Any scrollable list (commit list, file history, decisions list) must have its **header/toolbar OUTSIDE the scroll region** (sticky non-scrolling), and the scroll region gets `pt-0` with the first row fully visible. No content may be clipped at the top edge.
- Virtualized lists keep `@tanstack/react-virtual`; give the viewport an explicit height (`min-h-[24rem]`, grows to fill pane via flex) and `overflow-y-auto`. Add a subtle top/bottom fade or a persistent header divider so the boundary reads clearly.

### Interactive states (apply uniformly to tabs, list rows, buttons, timeline items)
| State | Treatment |
|-------|-----------|
| Default | Base surface; text at role color. |
| Hover | Surface lightens one step (`hover:bg-slate-800`); text brightens (`hover:text-slate-100`); transition `transition-colors`. |
| Selected/active | Left accent border or accent ring (`ring-1 ring-sky-500`) + `bg-slate-800`; accent text where it's a tab. Never rely on color alone — pair with border/ring or `aria-current`/`aria-selected`. |
| Focus (keyboard) | `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-sky-400 focus-visible:ring-offset-2 focus-visible:ring-offset-slate-950`. Every interactive element must have a visible focus ring. |
| Disabled | `opacity-50 cursor-not-allowed`, no hover change. |

### Primary tab nav (fixes the "tab labels wrap to two lines" defect)
- Horizontal, full-width bar under the header. Tabs laid out with `flex` and `gap-1`, each tab `whitespace-nowrap px-4 py-2` so labels like "Open work" never wrap.
- Active tab: `text-slate-100` + a 2px `sky-400` bottom border (underline style) OR filled `bg-slate-800` pill — pick one and apply consistently; underline preferred for the top-level nav.
- Inactive: `text-slate-400 hover:text-slate-200`.
- Set `role="tablist"` / `role="tab"` / `aria-selected` and roving focus; keep the existing `onIntelligenceTabChange` handler.

### Badges (attribution, confidence, open-work, repo)
- Pill shape `rounded-full`, `px-2 py-0.5`, Label size (12px), `border`. Use the semantic-state color table. `inline-flex items-center gap-1`.

## Era Timeline Label Strategy (fixes the core overlap/truncation defect)

The current `formatEraLabel` writes the full era name into each vis-timeline `content`, so adjacent range items collide into unreadable strings (e.g. "Mono Git Schema Ba Plugin distribution & onb"). Contract for the redesign — **CSS + label formatting only, no vis-timeline behavior change, no new dependency**:

1. **Constrain each item label box.** Via `.era-timeline .vis-item .vis-item-content`: set `max-width` per item, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap`, and comfortable inner padding (`0 8px`). A truncated label ends in `…` cleanly instead of bleeding into the neighbor.
2. **Full name always reachable.** Set the item `title` (native tooltip) to the full era name so hover reveals the untruncated label. The selected era's full name is also shown in the Era detail card in the rail — so truncation never hides information.
3. **Give the strip vertical room.** Era timeline card min-height `min-h-[8rem]` (up from the cramped current strip). If labels still collide horizontally at realistic era counts (≤ 8 named eras, per the 8-era cap in project decisions), rely on ellipsis + tooltip rather than shrinking the strip.
4. **Readable axis + item text.** Axis labels `slate-400` at 12px; item text `sky-100`/`sky-50` on the existing translucent cyan fill — keep the current selected/unselected fill treatment but ensure text contrast ≥ 3:1.
5. **Open-work badge inside items** keeps the existing `.open-work-badge` pattern (already styled) but must not push the label into overlap — it counts against the item's `max-width` budget, and the era name truncates first.
6. **Legibility target for the checker:** at 8 eras across a typical viewport, no label may render as run-together words from two different items; every item shows either its full name or a clean `…` ellipsis with a working hover tooltip.

Constraint: this is a `.era-timeline` CSS override + `title`/`content` formatting change in the existing component. Do not swap vis-timeline for another library.

## Copywriting Contract

Copy is diagnostic and evidence-oriented (product's job is evidence-backed answers). Verbs are concrete; empty/error states always name the next action.

| Element | Copy |
|---------|------|
| Primary CTA (commit list) | "Load more commits" |
| Primary CTA (tour player) | "Next stop" (secondary: "Previous stop"); start action: "Start tour" |
| Primary CTA (evidence drill) | "View commit" / "Open diff" / "Open file history" |
| Empty state — no index heading | "No analysis yet" |
| Empty state — no index body | "Run `/gitchange` in your IDE, or `gitchange index` in this repo, then reload to explore its history." |
| Empty state — no eras | "No eras detected yet — run semantic synthesis to name this project's chapters." |
| Empty state — no decisions | "No recorded decisions found for this repo." (matches EVD-03 honest-gap contract — never fabricate) |
| Empty state — no open work | "No open threads detected — no in-flight migrations or stale work found in the index." |
| Empty state — Decisions pane (nothing selected) | "Select a decision to view its evidence and drill into commits." |
| Empty state — Open work pane (nothing selected) | "Select a thread to view its migration timeline and drill into commits." |
| Error state — snapshot load failed | "Couldn't load the index. Confirm `.gitchange/` exists and re-run `gitchange index`, then reload." |
| Error state — graph missing | "Temporal graph not available yet. Run semantic synthesis to build `temporal-graph.json`." |
| Loading states | "Loading index status…" / "Loading commit history…" / "Loading project timeline…" / "Loading temporal graph…" (keep existing verb+noun form) |

**Destructive actions:** none in this phase. The dashboard is read-only over pre-built artifacts (SCALE-02); there are no delete/reset/write actions in any view. No destructive confirmations required. If a future control writes to the index, it is out of scope for this presentational redesign.

## Accessibility Contract

- All interactive elements reachable and operable by keyboard with the visible `sky-400` focus ring specified above.
- Tab nav uses `role="tablist"`/`tab`/`aria-selected`; panels use `aria-current` where appropriate (existing `aria-current` usage preserved).
- Error surfaces keep `role="alert"`.
- State is never conveyed by color alone — badges carry text labels; selected rows carry a border/ring in addition to background.
- Contrast: body/meta text ≥ 4.5:1, badge/label ≥ 3:1 (see Color section).
