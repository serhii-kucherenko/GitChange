<interfaces>
CommitListFilters (extend in commits.ts):
- author?: string — match authors.email or authors.name (case-insensitive LIKE)
- path?: string — EXISTS file_changes WHERE path LIKE path% for commit
- q?: string — message/summary LIKE %q%
- after?: number — committedAt >= after (unix seconds)
- before?: number — committedAt <= before

GET /api/commits query params mirror filters; invalid dates return 400.

CommitList uses useInfiniteQuery with filters in queryKey so filter change resets pagination.
</interfaces>
</context>
